# 一、产品是什么

一款面向快团团团长与经销商的食品饮料爆款文案生成工具。
用户输入产品基本信息，系统自动生成符合快团团开团格式的爆款标题与卖点文案，帮助团长和经销商告别手写文案，提升开团效率。

---

## 二、为什么要做这个产品

**现状与痛点**
团长和经销商在新品上架前需要手动撰写文案，耗时且质量不稳定。虽然市面上有通用AI工具可以辅助，
但存在两个核心问题：
- 生成的文案不符合快团团的格式规范，需要大量二次加工
- 没有结构化的输入引导，用户不知道该给AI提供哪些信息，导致生成质量参差不齐

**我们的解法**
通过结构化的产品信息录入表单+基于真实爆款案例训练的Prompt，一键生成直接可用的快团团标准格式文案，
让团长做到"填完即生成，生成即可用"。

---

## 三、用户是谁

**主要用户**
- 快团团团长
- 休闲食品/饮料经销商

**使用场景**
产品上架前，需要为快团团开团链接撰写标题和卖点描述文案。

**用户核心诉求**
1. 标题要吸睛，能在3秒内抓住社群用户注意力
2. 格式要规范，符合快团团文案的书写结构
3. 卖点描述要真实贴合产品本身，不能生硬夸大
4. 操作要简单，团长无需学习成本即可上手

---

## 四、用户使用流程

第一步：打开页面，看到欢迎语和产品介绍，
        点击"开始生成"进入表单

第二步：填写产品信息
  - 商品名（必填）
  - 品类，二级下拉选择（必填）
  - 口味/规格（必填）
  - 核心卖点标签，多选，选完品类后动态更新（必填，最多5个）
  - 团购价/原价（选填）
  - 目标人群，如宝妈/上班族/学生（选填）
  - 活动类型，如日常团/节日团/清仓特卖（选填）
  - 补充说明（选填）

第三步：选择文案风格（种草风/促销风/节日风）

第四步：点击“一键生成”，等待约 15-20 秒（页面显示进度提示：构思标题 → 提炼卖点 → 优化排版）

第五步：查看生成结果
  - 活动标题（1条，30字以内）
  - 商品标题（1条，8-30字）
  - 卖点短句（3条，每条简洁有力）
  - 卖点详细描述（3-5段，总计500字以内）
  - 卖点标签（3-5个关键词）

第六步：一键复制，直接粘贴至快团团开团页面对应字段

## 六、AI技术方案

### 模型选型与成本

| 模块 | 模型 | 选型理由 | 单次成本估算 |
|:---|:---|:---|:---|
| 文案生成 | deepseek-chat | 中文营销文案能力强，成本为GPT-4的1/10 | ￥0.003-0.005/次 |
| 合规审查 | 规则引擎（正则匹配） | 绝对化用语/敏感词检测无需模型，降低延迟 | ￥0 |

*注：按日均100次生成计算，月成本约￥10-15，完全可控。*

### Prompt工程架构
采用"系统人设 + 动态变量 + 品类化Few-shot"三层结构：

**System Prompt核心要素：**
- **角色设定**：带货超10万单的食品饮料专家（非通用AI，具备垂直领域销售经验）
- **语言风格**：口语化、真实、有感染力，严格禁止说明书体
- **格式约束**：强制JSON输出，包含5个标准字段，字数严格限制

**动态变量映射：**

| 用户输入 | Prompt变量 | 注入位置 |
|:---|:---|:---|
| 商品名+规格 | `{{product_info}}` | 标题生成模块 |
| 核心卖点标签 | `{{selling_tags}}` | 卖点展开模块 |
| 风格选择 | `{{style}}` | 语气调整模块（种草/促销/节日） |
| 目标人群 | `{{target_audience}}` | 场景化描述模块 |

**输出格式强制：**
```json
{
  "activity_title": "活动标题，30字内",
  "product_title": "商品标题，8-30字", 
  "selling_points_short": ["卖点1", "卖点2", "卖点3"],
  "selling_points_detail": "详细描述，500字内，3-5段",
  "tags": ["标签1", "标签2", "标签3"]
} 
```

### 异常处理与降级策略

| 异常场景 | 处理逻辑 | 用户感知 |
|:---|:---|:---|
| API超时（>20s） | 返回基于模板的保底文案 | "生成完成（极速版）" |
| 返回格式异常（非JSON） | `normalize_output`正则提取关键字段 | 正常展示 |
| 字段缺失 | 用输入信息智能补全（如商品名作为标题） | 正常展示 |
| 内容违规（敏感词） | 实时拦截并提示修改建议 | "请调整产品描述后重试" |

## 七、数据策略（Few-shot工程）

### 案例库规模

数据来源与规模：
- 当前规模：18条精选爆款案例（covering 17个二级品类）
- 来源：公开快团团爆款链接 + 基于真实高转化文案的人工改写
- 存储：本地examples.json，Git版本化管理，便于迭代

案例结构：
{
  "category_l1": "饮料",
  "category_l2": "咖啡饮料",
  "product_name": "示例商品名",
  "input_features": ["0糖", "独立小包装"],
  "style": "种草风",
  "output": {
    "activity_title": "...",
    "product_title": "...",
    "selling_points_short": ["...", "...", "..."],
    "selling_points_detail": "...",
    "tags": ["...", "..."]
  }
}

### 动态匹配策略

匹配逻辑（优先级排序）：
二级品类精准匹配：如用户选"咖啡饮料"，优先选用挂耳/即饮咖啡案例
一级品类兜底：无二级品类匹配时，动态获取当前用户选择的一级品类通用案例
风格交叉筛选：在品类匹配基础上，优先选用相同风格（种草/促销/节日）的案例
数量控制：选取TOP 2最相关案例注入Prompt，避免Token浪费

Fallback机制：
当匹配案例不足2条时，自动补充该一级品类下的高通用性案例，确保Few-shot始终有效。

### 数据迭代规划

| 阶段 | 目标 | 动作 |
|:---|:---|:---|
| **冷启动**（当前） | 18条覆盖主要品类 | 基于咖啡饮料深度打磨3-5条精品案例 |
| **MVP验证**（1个月内） | 扩充至50条 | 根据用户"重新生成"频率，识别低质量品类，针对性补充案例 |
| **优化期**（3个月内） | 建立质量评级 | 根据生成后的用户编辑率（二期追踪），淘汰高编辑率案例，保留"生成即用"的高分案例 |

## 十、合规要求

### 广告法合规
- 绝对化用语过滤：禁用"最健康"、"第一"、"纯天然"、"100%有效"等词汇（正则匹配拦截）
- 健康功能保守化：避免医疗暗示，如"减肥"改为"控卡"、"排毒"改为"膳食纤维"、"治疗"改为"舒缓"
- 功效承诺限制：不使用"立竿见影"、"绝对有效"、"治愈"等确定性承诺

### 平台规范合规

- 禁用平台违禁词：如"微商"、"代理"、"刷单"等快团团敏感词
- 价格表述规范：如有比价，需真实（原价需为真实售价，非虚构高价）
- 过敏原信息：如用户选择含坚果/乳制品/麸质等标签，必须在文案中体现过敏原提示

### 内容风格合规（避免说明书体）

**避免生硬表述：**
- ❌ "保质期：12个月" → ✅"赏味期长达12个月，囤货无压力"
- ❌ "净含量500ml" → ✅"大瓶装500ml，刚好够喝一下午"  
- ❌ "配料表：..." → ✅"配料表很干净，只有..."

**检测机制：**
- 命中"保质期："、"净含量："等**冒号+数字**的说明书格式 → 提示优化
- 口语化表达"保质期长达"、"一大瓶500ml" → 通过

检测与重写机制：
- Prompt层预防：System Prompt中明确禁止上述词汇，并提供替换示例（如"保质期12个月"→"赏味期长达12个月"）
- 后置检测：normalize_output函数检测说明书体词汇，命中则触发重写或标记提示
- 人工兜底：极端情况下返回模板化安全文案，确保用户可立即使用

## 十一、不做什么（本期范围外）

- 不支持图片/视频等素材生成，仅限文字文案
- 不做批量导入产品，本期只支持单条录入
- 不扩展至快团团以外的其他平台
- 用户登录与历史记录为二期功能
